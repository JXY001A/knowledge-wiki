> Claude Code 的内置文件系统记忆机制，按项目隔离、分类型存储，实现跨会话持久化。

## 是什么

Claude Code 长期记忆是一套基于 markdown 文件的持久化记忆系统。它在文件系统中以结构化目录存储用户偏好、反馈、项目背景和外部资源指针，每次新会话启动时自动加载，使 Claude Code 在不同会话间保持一致的协作风格和上下文认知。

## 架构

```
~/.claude/
├── memory/                              ← 全局记忆层（所有项目共享）
│   ├── MEMORY.md                        ← 全局索引（会话启动时加载）
│   ├── feedback-reply-in-chinese.md
│   └── feedback-explain-before-execute.md
│
└── projects/
    └── -Users-jinxianyu-code-knowledge-wiki/  ← 项目记忆层（按工作目录隔离）
        └── memory/
            ├── MEMORY.md                ← 项目索引（会话启动时加载）
            └── dev-machine-connection.md
```

### 两层设计

| 层级 | 路径 | 作用 | 示例 |
|------|------|------|------|
| 全局 | `~/.claude/memory/` | 用户级别的偏好和规则，所有项目通用 | "用中文回复"、"执行前先解释" |
| 项目 | `~/.claude/projects/<路径>/memory/` | 特定项目的上下文、资源、约束 | DevMechin 连接信息、项目决策 |

项目路径由工作目录转换而来（`/` 替换为 `-`），不同项目之间完全隔离。

## 加载机制

```
会话启动
    │
    ├── 1. 读取 ~/.claude/memory/MEMORY.md（如果存在）
    │       └── 加载全局索引，≤200 行
    │
    ├── 2. 读取 ~/.claude/projects/<项目>/memory/MEMORY.md
    │       └── 加载项目索引，≤200 行
    │
    └── 3. 按需读取——根据对话内容，匹配具体 .md 记忆文件的 description 字段
            └── 只加载相关的，不全量塞入上下文
```

## 记忆类型

| 类型 | 用途 | 写入时机 | 示例 |
|------|------|---------|------|
| `user` | 用户角色、知识背景、偏好 | 了解到用户的角色或偏好时 | "用户是数据科学家" |
| `feedback` | 行为纠正和确认 | 用户纠正或确认非常规做法 | "用中文回复"、"先解释再执行" |
| `project` | 项目背景、决策、约束 | 了解到项目层面的 who/what/why/when | "下周四起冻结非关键合并" |
| `reference` | 外部系统指针 | 了解到外部资源的地址和用途 | DevMechin 连接方式、Grafana 面板地址 |

## 优先级关系

全局与项目记忆**不存在技术上的优先级覆盖**——两者都会被加载到同一上下文中，互补而非互斥。

当出现矛盾时，遵循惯例：

```
项目级 > 全局级         （更具体的优先）
用户明确指令 > 记忆     （会话中明确说的覆盖记忆）
当前代码 > 记忆          （代码现状优先于记忆中的快照）
```

实际使用时，全局记忆放通用规则，项目记忆放项目专属信息——天然的关注点分离，很少产生冲突。

## 写入规则

### 应该写入
- 用户角色、偏好、习惯
- 用户纠正或确认的行为规范
- 项目决策、约束、时间节点
- 外部系统地址和用途

### 不应该写入
- 代码模式、架构、文件路径（应读代码获取）
- git 历史（`git log` 是权威）
- 临时任务和进行中工作
- CLAUDE.md 已有的内容
- 密码、Token 等敏感信息

### 写入流程

```
1. 创建具体 .md 文件（按类型命名，如 feedback-xxx.md）
2. 在 MEMORY.md 中添加一行索引（≤150 字概要）
3. 检查是否有重复或过期记忆需要更新
```

## 与其他持久化机制的边界

| 机制 | 用途 | 生命周期 |
|------|------|---------|
| 长期记忆 | 跨会话协作风格、外部资源定位 | 持久，手动管理 |
| CLAUDE.md | 项目级的代码规范和架构指导 | 随 git 版本控制 |
| AGENTS.md | wiki Schema 定义 | 随 git 版本控制 |
| 知识库（wiki/） | 领域知识、资料摘要 | 随 git 版本控制 |
| 计划/任务 | 当前会话的工作分解 | 会话内 |

## 常见误解

- **“记忆就是知识库”** — 不是。记忆存的是"怎么和你协作"，知识库存的是"领域知识"。
- **“MEMORY.md 写入具体的记忆内容”** — 不是。MEMORY.md 只是索引，具体内容在独立 `.md` 文件中。
- **“全局记忆覆盖项目记忆”** — 不是。两者互补，不存在层级覆盖。
