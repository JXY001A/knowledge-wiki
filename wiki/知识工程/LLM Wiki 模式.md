> Andrej Karpathy 提出的个人知识库构建模式：LLM 渐进式维护结构化 Wiki，而非每次查询时从原始文档重新检索。知识被"编译"一次，保持更新，而非每次"解释"。

## 背景与动机

传统 RAG 在每次查询时重新检索原始文档片段，知识无法累积。LLM Wiki 的核心创新在于：让 LLM 增量式构建和维护一个持久化 Wiki——结构化的、相互链接的 Markdown 文件集合，介于用户与原始资料之间。每次添加新资料时，LLM 不只是索引用以检索，而是深度处理：提取关键信息、更新实体页、修正主题摘要、标注矛盾、强化综合分析。

## 架构

三层结构：

| 层 | 职责 | 谁操作 |
|------|------|--------|
| `raw/` | 不可变源文档 | 人类只读 |
| `wiki/` | LLM 生成的 Markdown 页面 | LLM 完全拥有 |
| `AGENTS.md` | Schema 规范，定义结构和约定 | 人机共演 |

## 三个操作

```
Ingest（摄入）→ Query（查询）→ Lint（巡检）
     ↓                ↓               ↓
  深度处理         综合回答         健康检查
  更新10-15页      可归档为新页      修复问题
```

## 为什么有效

维护知识库的繁琐部分不是阅读或思考，而是"记账"——更新交叉引用、保持摘要最新、维护一致性。LLM 不会觉得无聊，不会忘记更新，可以一次性处理 15 个文件。维护成本接近零。

## 适用场景

- 个人深度研究
- 结构化读书笔记
- 项目知识管理（PKM）
- AI Agent 记忆固化
- 小型团队内部 Wiki

## 局限

- 无数据库依赖，搜索靠 index.md + grep
- 规模天花板：数百~低千页后性能下降
- 无自动化调度（需手动触发）
- wikilink 缺乏类型化边
