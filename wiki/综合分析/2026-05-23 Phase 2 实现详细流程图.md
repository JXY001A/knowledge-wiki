## 二、企业微信消息处理全链路

```mermaid
flowchart TD
    WX["手机微信"] -->|"用户发消息"| WXServer["企业微信服务端"]
    WXServer -->|"POST XML（AES 加密）"| Frps["阿里云 frps | 8.133.175.201:9400"]
    Frps -->|"frp 隧道"| Frpc["DevMechin frpc | forward to 127.0.0.1:9400"]
    Frpc -->|"转发"| Flask["Flask receive_message"]
    Flask -->|"1. 提取加密 XML"| Decrypt["decrypt_msg | AES-256-CBC 解密"]
    Decrypt -->|"2. 明文 XML"| Parse["解析 MsgType | FromUserName | Content"]
    Parse -->|"3. 后台线程"| Thread["threading.Thread | target=process_message"]
    Thread --> Dispatch["process_message | Skill 路由引擎"]
    Dispatch -->|"? + 搜索"| SkillSearch["search-wiki"]
    Dispatch -->|"? + 查询"| SkillQuery["query-knowledge"]
    Dispatch -->|"URL"| SkillIngest["ingest-article"]
    Dispatch -->|"检查 lint"| SkillLint["lint-wiki"]
    Dispatch -->|"其他文本"| SkillSave["save-note（默认回退）"]
    SkillSearch --> Reply["send_markdown | to 企业微信 API"]
    SkillQuery --> ReplyCards["send_template_card | 多卡片 + markdown"]
    SkillIngest --> Reply
    SkillLint --> Reply
    SkillSave --> Reply
    Reply --> WXAPI["qyapi.weixin.qq.com | 主动消息 API"]
    ReplyCards --> WXAPI
    WXAPI -->|"推送"| WX
```

## 三、Skill 执行引擎内部流程

```mermaid
flowchart TD
    Start["process_message(user_id, text, cb_md, cb_tpl)"] --> Strip["stripped = text.strip()"]
    Strip --> BuildCtx["构建 context 字典 | user_id, input_text | send_md, send_tpl callbacks"]
    BuildCtx --> Check{"text 以 ? 开头?"}
    Check -->|"是"| ExtractQ["question = stripped[1:] | ctx['query'] = question"]
    ExtractQ --> MatchQ["match_skill(question)"]
    Check -->|"否"| CheckURL{"is_url(text)?"}
    CheckURL -->|"是"| MatchURL["match_skill(url)"]
    CheckURL -->|"否"| MatchText["match_skill(text)"]
    MatchQ --> Found{"skill 匹配成功?"}
    MatchURL --> Found
    MatchText --> Found
    Found -->|"是"| Exec["execute_skill(skill.name, ctx)"]
    Found -->|"否"| Fallback{"回退链"}
    Fallback -->|"?"| FBQuery["execute_skill('query-knowledge', ctx)"]
    Fallback -->|"URL"| FBIngest["execute_skill('ingest-article', ctx)"]
    Fallback -->|"text"| FBSave["execute_skill('save-note', ctx)"]
    Exec --> LoadSkill["1) find_skill(name) | → Skill 对象"]
    FBQuery --> LoadSkill
    FBIngest --> LoadSkill
    FBSave --> LoadSkill
    LoadSkill --> LoadImpl["2) _load_impl(skill) | importlib 加载 | skills/{name}/impl.py"]
    LoadImpl --> HasImpl{"impl.py 存在?"}
    HasImpl -->|"是"| CallImpl["3) impl.execute(context)"]
    HasImpl -->|"否"| FallbackMD["返回 SKILL.md 文本 | + 提示缺少实现"]
    CallImpl --> Async{"async def execute?"}
    Async -->|"是"| Await["asyncio.run(execute(ctx))"]
    Async -->|"否"| Sync["execute(ctx)"]
    Await --> Result["返回结果文本"]
    Sync --> Result
    FallbackMD --> Result
    Result --> SendBack{"结果非空?"}
    SendBack -->|"是"| SendMD["send_md(user_id, result[:3000])"]
    SendBack -->|"否（技能已回调发送）"| Done["处理完成"]
    SendMD --> Done
```

## 四、URL 摄取完整流水线（ingest-article）

```mermaid
flowchart TD
    URL["用户发送 URL"] --> Notify["1) 通知用户：正在提取并分析"]
    Notify --> Fetch["2) fetch_url_text: HTTP GET 下载网页"]

    FetchDetail --> FetchOK{"4. 下载成功?"}
    FetchOK -->|"no"| SaveRef["5a. 保存待摄取链接到收件箱，git push"]

    SaveRaw --> DeepSeek["6) call_ingest: 发送到 DeepSeek API"]
    DeepSeek --> DeepSeekDetail["7) POST deepseek-v4-pro，全量 ingest prompt，temp 0.1，max 4096 tokens"]
    DeepSeekDetail --> DeepSeekOK{"8. LLM 返回有效 JSON?"}
    DeepSeekOK -->|"no"| NotifyLLMFail["8a. 通知用户：LLM 分析失败，已保存原文"]
    DeepSeekOK -->|"yes"| ParseJSON["9) json.loads: 解析 title, domain, tags, summary, key_points, notes, quotes, concepts, related_pages"]
    ParseJSON --> BuildSource["10) build_source_page: 生成资料摘要页，写入 frontmatter 和正文"]

    AppendLog --> GitPush["13) commit_and_push: git add, commit, push"]

```

## 五、知识查询流水线（query-knowledge）

```mermaid
flowchart TD
    Q["用户: ? AI Workflow 是什么"] --> MCP["1) mcp_query: 通过 MCP 检索 wiki 内容"]
    MCP --> MCPDetail["2) MCP session: initialize, get Session-Id, tools/call query"]
    MCPDetail --> MCPOK{"3. 找到结果?"}
    MCPOK -->|"no"| SearchFallback["4a. 回退: tools/call search"]
    SearchFallback --> SearchFallbackOK{"4b. search 找到?"}
    SearchFallbackOK -->|"no"| NotFound["5a. send_md: 暂无相关内容"]
    SearchFallbackOK -->|"yes"| ReadPages["5a. 读取匹配页面全文，strip_frontmatter，限 2000 字"]
    MCPOK -->|"yes"| StripFM["4) strip_frontmatter: 去除 YAML 元数据"]
    ReadPages --> StripFM
    StripFM --> LLM["5) call_json: 调用 qwen2.5:3b 生成结构化卡片"]
    LLM --> LLMDetail["6) POST Ollama api/chat, num_predict 2000, temp 0.1, Output ONLY valid JSON"]
    LLMDetail --> JSONOK{"7. JSON 解析成功?"}
    JSONOK -->|"no"| KWSearch["8a. 回退: keyword_search 返回第一匹配页面"]
    KWSearch --> KWSend["8b. send_template_card: 第一匹配页面"]
    JSONOK -->|"yes"| ProcessCards["8) 遍历 cards 前 8 个"]
    ProcessCards --> ForEach["9) for each card: enumerate cards"]
    ForEach --> Sleep{"10. i 大于 0?"}
    Sleep -->|"yes"| Wait["10a. time.sleep 800ms 防止企业微信乱序"]
    Sleep -->|"no"| Normalize["10a. normalize_details: 标准化为键值对"]
    Wait --> Normalize
    Normalize --> ShortSummary{"11. card.summary 为空?"}
    ShortSummary -->|"yes"| CallShort["11a. call_short: qwen2.5:3b 生成一句话摘要"]
    ShortSummary -->|"no"| SendCard["11a. send_template_card: 发送卡片到企业微信"]
    CallShort --> SendCard
    SendCard --> MoreCards{"12. 还有卡片?"}
    MoreCards -->|"yes"| ForEach
    MoreCards -->|"no"| Overall{"13. card_data.summary 超 100 字?"}
    Overall -->|"yes"| SendOverall["14) send_markdown: 补充综合文字"]
    Overall -->|"no"| Done["完成"]
    SendOverall --> Done
```

**LLM 调用四级容错链**（内建于 `query-knowledge/impl.py`）：

```
qwen2.5:3b JSON
  └─ 失败 → keyword_search() 直接返回第一匹配页面
  └─ 成功 → 遍历 cards，每张 card.summary 为空时
              └─ qwen2.5:3b call_short() 一句话摘要
              └─ 失败 → 跳过该 card
多卡片发送后 → card_data.summary > 100 字时 send_markdown 补充
```

## 七、各阶段对比

| | Phase 0（最初） | Phase 1（基座升级） | Phase 2（当前） |
|---|---|---|---|
| **代码组织** | 3 个独立 repo，2 个单文件脚本 | 统一 `src/knowledge_wiki/` 包 | 不变 |
| **消息分发** | 硬编码 `if/elif/else` | 硬编码 `if/elif/else` | Skill 引擎路由 |
| **技能系统** | 无 | `skill/` 骨架（占位） | 5 技能 + 执行引擎 |
| **LLM 调用** | 散落各处，接口不一 | 统一 `llm/` 包 | 不变 |
| **部署** | systemd 明文密钥 | `.env` + `EnvironmentFile` | 不变 |
| **测试** | 0 | 20 项 | 32 项 |
| **可扩展性** | 加功能 = 加 if/elif 分支 | 加功能 = 加模块 | 加功能 = 加技能目录 |

## 八、模型资源清单

### 本地模型（DevMechin Ollama · RTX 4080 16GB）

| 模型 | 大小 | 用途 | 调用函数 | 参数特征 |
|------|------|------|---------|---------|
| `qwen2.5:3b` | 1.9 GB | **主查询模型** — JSON 结构化输出 | `call_json()` | num_predict=2000, temp=0.1 |
| `qwen2.5:3b` | — | 一句话摘要 | `call_short()` | num_predict=120, temp=0.1 |
| `qwen2.5:3b` | — | 详细 markdown 回答 | `call_detailed()` | num_predict=1500, temp=0.3 |
| `qwen2.5:1.5b` | 986 MB | （备用）轻量中文推理 | — | 已安装，当前未调用 |
| `llama3.2:1b` | 1.3 GB | **最终回退** — 所有模型失败时 | `call_fallback()` | num_predict=200, temp=0.1 |
| `qwen3:4b` | 2.5 GB | （备用）中文通用推理 | — | 已安装，当前未调用 |
| `qwen3-vl:8b` | 6.1 GB | 图片解析（`~/bin/vision`） | — | 独立脚本，非 MCP 链路 |

### 云端模型

| 模型 | 提供商 | 用途 | 调用函数 | 参数特征 |
|------|--------|------|---------|---------|
| `deepseek-v4-pro` | DeepSeek API | **ingest 分析** — URL → wiki 结构化数据 | `call_ingest()` | max_tokens=4096, temp=0.1 |

## 十一、模型选型决策表

| 决策因素 | qwen2.5:3b（主） | llama3.2:1b（回退） | deepseek-v4-pro |
|---------|------------------|--------------------|--------------------|
| **中文质量** | 优秀 | 一般 | 优秀 |
| **JSON 输出** | 较好（偶需修复） | 差 | 优秀 |
| **延迟** | 3-10s | 1-3s | 10-30s（网络） |
| **上下文** | 32K | 128K | 128K |
| **显存占用** | 1.9 GB | 1.3 GB | 0（云端） |
| **成本** | 免费 | 免费 | ¥0.01/篇 |
| **适用场景** | 查询 / 摘要 | 紧急回退 | 摄取分析 |
