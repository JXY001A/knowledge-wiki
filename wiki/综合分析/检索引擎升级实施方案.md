# 检索引擎升级实施方案

> 基于设计文档 v1.0 的落地分析，按优先级重新排序。Phase 3（图谱加权）暂缓，200 页后启动。

## 整体策略

**先拿最大收益，再逐步叠加。** 三个阶段按顺序实施，每完成一个即可上线验证。

```
Phase 1+2（分词 + BM25）→ 上线验证 → Phase 4（分层组装）→ 上线验证 → Phase 3（图谱，200页后）
```

## 核心判断

| 阶段 | 收益占比 | 开发量 | 上线后可立即看到效果 |
|------|:---:|------|:---:|
| Phase 2（BM25 + jieba） | **~60%** | 2 天 | 是 |
| Phase 4（分层组装） | ~25%（LLM 作答质量） | 2 天 | 是 |
| Phase 1（Catalog 初筛） | ~15%（性能为主） | 1 天 | 是 |
| Phase 3（图谱加权） | 当前～0%（50页），200页后 ~15% | 2 天 | 否 |

### tokenizer.py

```python
# 核心接口
def tokenize(text: str, mode: Literal["search", "index"] = "search") -> list[str]
def remove_stopwords(tokens: list[str]) -> list[str]
def load_custom_dict(dict_path: Path) -> None
```

**分词策略**：
- `mode="search"`：`jieba.cut_for_search`，细粒度（提高召回）
- `mode="index"`：`jieba.lcut`，粗粒度（保留语义单元）
- 自定义词典：从所有 wiki 页面 title 动态生成，每次 lint 刷新
- 停用词：中文"什么/怎么/如何/是/的/了" + 英文"the/a/an/of/in"

### bm25.py

```python
# 核心接口
def build_bm25_index(wiki_dir: Path) -> BM25Index
def bm25_search(
    index: BM25Index,
    query: str,
    top_n: int = 10,
) -> list[BM25Result]
```

**多字段加权公式**：
```
final_score = 3.0 × BM25(title) + 2.0 × BM25(tags) + 1.0 × BM25(body)
```

| 字段 | 权重 | 理由 |
|------|:---:|------|
| title | 3.0 | 命中标题是最强信号 |
| tags | 2.0 | 受控词汇，命中即高相关 |
| body | 1.0 | 正文兜底 |

**BM25 参数**：

| 参数 | 默认 | 说明 |
|------|:---:|------|
| `k1` | 1.5 | 词频饱和点 |
| `b` | 0.75 | 长度归一化 |

### index_store.py

```python
# 核心接口
def save_index(index: BM25Index, path: Path) -> None
def load_index(path: Path) -> BM25Index | None
def index_is_fresh(index: BM25Index, wiki_dir: Path) -> bool
```

**序列化位置**：`.cache/retrieval/bm25.pkl.zst`（zstd 压缩 pickle，不入 git）

**启动校验流程**：
```
1. 检查 .cache/retrieval/bm25.pkl.zst 是否存在
2. 反序列化，校验 version == INDEX_VERSION
3. 校验 num_docs == 当前 wiki/ 下 .md 数量
4. 任一校验失败 → 全量重建
```

**索引重建触发点**：

| 触发点 | 操作 |
|--------|------|
| 服务启动 | load → 失败/过期 → 全量重建 |
| ingest 完成 | 增量更新 doc_meta；累计 ≥ 5 个变更 → 全量重建 |
| lint 完成 | 强制全量重建（可能改了大量文件） |
| 手动 CLI `retrieval rebuild` | 强制全量重建 |

## Phase 2：Catalog 目录初筛（Day 3）

### 目标

用 `Wiki 目录.md` 做廉价初筛，把 BM25 候选集从全量（未来 500+ 页）缩减到 20 页，同时提供类别意图信号。P95 延迟从可能超过 300ms 控制到 <100ms。

### 新增文件

| 文件 | 说明 |
|------|------|
| `src/knowledge_wiki/wiki/retrieval/catalog.py` | 目录解析与初筛 |
| `tests/retrieval/test_catalog.py` | 目录解析单元测试 |

### catalog.py

```python
# 核心接口
def parse_catalog(catalog_path: Path) -> CatalogIndex
def catalog_filter(query_tokens: list[str], index: CatalogIndex, k: int = 20) -> list[str]
```

**CatalogIndex 数据结构**：
```python
CatalogIndex = {
    "categories": {
        "AI平台": ["DeepSeek", "Ollama", ...],
        "概念": ["AI Workflow", ...],
        "综合分析": ["知识库系统全链路架构", ...],
    },
    "all_titles": {"DeepSeek", "Ollama", ...},  # set, O(1) 查找
}
```

**解析规则**（从 `Wiki 目录.md` 正则提取）：
```python
# H2 → 大类（领域页、概念、综合分析等）
# H3 → 子类（AI平台、工程化等）
# - [[PageName]] → 页面项
```

**初筛算法**：
```
1. 遍历 all_titles，title 命中查询 token 即算分
2. 类别 boost：
   - 查询含 "对比"/"vs" → "综合分析" 类别 +2
   - 查询含 "什么"/"是什么" → "概念" 类别 +1
3. 取 Top-K (K=20)
4. 若结果 < 10，回退全量（避免初筛过严）
```

**关键参数**：

| 参数 | 默认 | 说明 |
|------|:---:|------|
| `CATALOG_TOP_K` | 20 | 初筛候选集大小 |
| `CATALOG_FALLBACK_THRESHOLD` | 10 | 不足此数时回退全量 |

### 与 BM25 Pipeline 集成

```python
# pipeline.py 中
candidates = catalog_filter(tokens, catalog, k=20)
bm25_results = bm25_search(index, query, candidates=candidates, top_n=10)
```

`candidates` 传入 BM25 搜索，BM25 仅对候选集内文档打分，大比例降低计算量。

## Phase 3：分层组装（Day 4-5）

### 目标

把当前"Top-8 × 800 字一锅炖"改为三层结构化输出，让 LLM 从"被噪音淹没"变成"先看结论 → 再看引用 → 再看证据"。不直接影响 P@5，但显著提升 LLM 作答质量。

### 新增文件

| 文件 | 说明 |
|------|------|
| `src/knowledge_wiki/wiki/retrieval/layered.py` | 三层组装逻辑 |
| `src/knowledge_wiki/wiki/retrieval/pipeline.py` | 端到端流水线编排（此阶段引入） |
| `tests/retrieval/test_layered.py` | 分层组装单元测试 |

### 三层结构

```
┌─────────────────────────────────────────────┐
│ Layer 1: 编译真相                            │
│ Top-1 文档的 YAML frontmatter + 正文前 400 字 │
│ 预算: ≤ 1500 tokens                         │
│ 作用: LLM 第一眼看到结论性信息               │
└─────────────────────────────────────────────┘

┌─────────────────────────────────────────────┐
│ Layer 2: 交叉引用                            │
│ Top-1 文档中提到的所有 [[wikilink]] 邻居      │
│ 每个邻居: title + 一句话摘要                 │
│ 预算: ≤ 1000 tokens                         │
│ 作用: 告诉 LLM 有哪些相关页面可深入          │
└─────────────────────────────────────────────┘

┌─────────────────────────────────────────────┐
│ Layer 3: 证据补充                            │
│ Top-2..N 文档全文（按 BM25 score 降序）       │
│ 操作日志中相关条目                           │
│ 预算: 剩余约 5500 tokens                    │
│ 作用: LLM 验证、补充、引用                   │
└─────────────────────────────────────────────┘
```

**Token 总预算：8000**（适配 32K context 模型，预留 75% 给对话历史和系统指令）

### 一句话摘要提取

Layer 2 中每个邻居的摘要，按优先级：
1. 正文第一段第一句（`split("。")[0]`）
2. 标题（fallback）

### 反哺钩子

分层组装末尾附带机器可读 footer：

```yaml
# retrieval_footer
top_n_titles: [title1, title2, ...]
candidates_considered: N
suggest_create_synthesis_page: true/false
```

`suggest_create_synthesis_page: true` 时，LLM 主动建议建综合分析页。

### 流水线集成

```python
# pipeline.py 端到端入口
def run_pipeline(query: str, top_n: int = 5) -> str:
    # Phase 0: 预处理
    tokens = remove_stopwords(tokenize(query, mode="search"))
    
    # Phase 1: Catalog 初筛
    catalog = parse_catalog("wiki/Wiki 目录.md")
    candidates = catalog_filter(tokens, catalog, k=20)
    
    # Phase 2: BM25 排序
    index = load_or_build_index()
    bm25_results = bm25_search(index, query, candidates, top_n=10)
    
    # Phase 3: 分层组装
    return assemble_layered(query, bm25_results[:top_n])
```

## 回滚方案

任一阶段出问题，环境变量立即切回旧实现：

```bash
KNOWLEDGE_WIKI_USE_NEW_RETRIEVAL=false
```

旧 `query_tool()` 代码完整保留在 `_query_tool_legacy()` 中，随时可回。

## 风险与缓解

| 风险 | 概率 | 缓解 |
|------|:---:|------|
| jieba 技术新词分词差 | 高 | 自定义词典 + lint 时动态补充 |
| BM25 同义词召回为零 | 中 | 200 页后加向量检索（v2.0） |
| 索引文件损坏 | 低 | 启动校验 + 失败自动重建 |
| 索引与文件不同步 | 中 | ingest/lint 后增量或全量重建 |

## 暂缓：Phase 4 图谱加权（200 页后）

**暂缓原因**：
1. 当前 50+ 页面，wikilink 链接稀疏（大多页面只有 2-5 个入链）
2. 入度 boost 在稀疏图中反而可能让少数 hub 页霸榜，降低多样性
3. 邻居扩展在图稀疏时拉进来的多是弱相关页面
4. 预计 200 页时（约 3-6 个月后）图密度足够支撑有意义的权重计算

**启动条件**：
- wiki 页面 ≥ 200
- 平均每页入链 ≥ 5
- 黄金集标注完成（≥ 30 对）

**启动时参数建议**：

| 阶段 | α (入度) | β (标签) | γ (邻居) |
|------|:---:|:---:|:---:|
| 刚启动（200-300 页） | 0.1 | 0.3 | 0.1 |
| 成熟期（300-500 页） | 0.3 | 0.4 | 0.2 |
| 大规模（500+ 页） | 0.4 | 0.4 | 0.3 |
